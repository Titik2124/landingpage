# PRD - Chatime Landing Page

## 1. Ringkasan Produk

### 1.1 Nama Produk

Chatime Indonesia Landing Page

### 1.2 Latar Belakang

Chatime membutuhkan landing page yang modern, responsif, dan mudah digunakan untuk memperkuat brand awareness, menampilkan menu utama, mengomunikasikan promo, membantu pengguna menemukan outlet terdekat, serta mengarahkan pengguna ke kanal pemesanan.

Dokumen ini mengasumsikan landing page dibuat untuk kebutuhan resmi/terotorisasi Chatime. Semua logo, foto produk, nama menu, harga, promo, dan aset brand harus menggunakan materi yang sudah memiliki izin penggunaan.

### 1.3 Tujuan Produk

- Memperkenalkan Chatime sebagai brand minuman teh dan bubble tea yang mudah diakses.
- Membantu pengguna melihat menu, promo, dan produk unggulan dengan cepat.
- Mengarahkan pengguna ke aksi utama: pesan sekarang atau cari outlet.
- Menyediakan halaman yang cepat, mobile-first, SEO-friendly, dan mudah dikembangkan.

### 1.4 Target Pengguna

- Remaja dan dewasa muda usia 15-35 tahun.
- Pengguna mobile di kota besar dan area urban.
- Pelanggan yang ingin melihat menu/promo sebelum membeli.
- Pelanggan yang ingin mencari outlet terdekat.
- Calon mitra atau pihak yang ingin menghubungi brand.

## 2. Problem Statement

Pengguna sering membutuhkan informasi cepat tentang menu, promo, lokasi outlet, dan cara membeli produk Chatime. Jika informasi tersebar di banyak kanal atau sulit diakses melalui mobile, pengguna berpotensi batal membeli atau berpindah ke kompetitor.

Landing page harus menyatukan informasi penting tersebut dalam pengalaman yang visual, cepat, dan jelas.

## 3. Sasaran dan Metrik Sukses

### 3.1 Business Goals

- Meningkatkan klik ke kanal pemesanan online.
- Meningkatkan pencarian outlet.
- Meningkatkan awareness terhadap menu dan promo baru.
- Membangun fondasi digital yang bisa dikembangkan ke campaign berikutnya.

### 3.2 Product Metrics

| Metrik | Target Awal |
| --- | --- |
| Lighthouse Performance | >= 90 |
| Lighthouse Accessibility | >= 90 |
| Largest Contentful Paint | < 2.5 detik |
| Click-through rate tombol Order | >= 3% |
| Click-through rate Find Store | >= 5% |
| Bounce rate | < 45% |
| Average session duration | > 90 detik |

## 4. Scope Produk

### 4.1 In Scope

- Landing page satu halaman.
- Navbar responsif.
- Hero section dengan visual produk.
- Section menu produk.
- Section best seller.
- Section promo.
- Store locator sederhana.
- Section about brand.
- Section contact/franchise inquiry.
- Footer dengan link dan social media.
- SEO dasar, Open Graph, dan metadata.
- Responsive design untuk mobile, tablet, dan desktop.

### 4.2 Out of Scope

- Sistem checkout/e-commerce penuh.
- Login/register pengguna.
- Loyalty point atau membership.
- Dashboard admin.
- Integrasi pembayaran.
- Real-time inventory outlet.
- Multi-language.

## 5. User Persona

### 5.1 Pelanggan Baru

Pelanggan yang belum familiar dengan menu Chatime dan ingin melihat produk populer sebelum membeli.

Kebutuhan:

- Melihat produk unggulan.
- Memahami kategori menu.
- Mendapat rekomendasi best seller.

### 5.2 Pelanggan Rutin

Pelanggan yang sudah mengenal brand dan ingin mencari promo atau outlet terdekat.

Kebutuhan:

- Mengakses promo dengan cepat.
- Menemukan outlet terdekat.
- Masuk ke kanal order online.

### 5.3 Calon Mitra

Pengguna bisnis yang ingin menghubungi Chatime untuk kerja sama atau informasi kemitraan.

Kebutuhan:

- Melihat informasi brand secara ringkas.
- Mengisi formulir kontak.
- Mendapat jalur komunikasi resmi.

## 6. User Journey

### 6.1 Journey Utama: Order Produk

1. Pengguna membuka landing page.
2. Pengguna melihat hero dan CTA "Order Now".
3. Pengguna menelusuri menu atau best seller.
4. Pengguna memilih CTA order.
5. Pengguna diarahkan ke kanal pemesanan resmi atau partner delivery.

### 6.2 Journey Utama: Cari Outlet

1. Pengguna membuka landing page dari mobile.
2. Pengguna memilih CTA "Find Store".
3. Pengguna melihat input lokasi atau daftar outlet.
4. Pengguna memilih outlet.
5. Pengguna membuka peta/navigasi.

### 6.3 Journey Tambahan: Lihat Promo

1. Pengguna membuka halaman.
2. Pengguna melihat banner promo.
3. Pengguna membaca detail singkat promo.
4. Pengguna menekan CTA untuk order atau klaim promo.

## 7. Struktur Halaman

### 7.1 Navbar

Konten:

- Logo Chatime.
- Menu navigasi: Home, Menu, Promo, Outlet, About, Contact.
- Tombol utama: Order Now.
- Hamburger menu untuk mobile.

Perilaku:

- Sticky di bagian atas.
- Scroll ke section terkait saat menu diklik.
- Di mobile, menu terbuka sebagai drawer atau dropdown yang mudah ditutup.

### 7.2 Hero Section

Konten:

- Headline: "Make Every Sip Count"
- Subcopy: "Temukan minuman favoritmu dari milk tea klasik, fruit tea segar, hingga kreasi seasonal."
- CTA utama: Order Now.
- CTA sekunder: Find Store.
- Visual utama: foto produk Chatime atau komposisi produk best seller.

Kriteria:

- Produk harus terlihat jelas pada viewport pertama.
- CTA harus terlihat tanpa scroll pada mobile dan desktop.
- Hindari visual yang terlalu gelap, blur, atau stock-like.

### 7.3 Menu Section

Konten:

- Kategori: Milk Tea, Fruit Tea, Coffee, Smoothies, Tea Presso, Seasonal.
- Kartu produk berisi gambar, nama produk, kategori, deskripsi pendek, dan label best seller bila ada.
- Filter kategori.

Contoh produk placeholder:

- Pearl Milk Tea.
- Thai Tea.
- Matcha Milk Tea.
- Brown Sugar Milk Tea.
- Mango Green Tea.
- Lemon Black Tea.

Kriteria:

- Filter kategori tidak mengubah layout secara janggal.
- Gambar produk memiliki ukuran konsisten.
- Kartu produk tetap rapi di mobile.

### 7.4 Best Seller Section

Konten:

- 3 sampai 5 produk unggulan.
- Visual produk lebih besar.
- Deskripsi rasa singkat.
- CTA "Lihat Menu".

Tujuan:

- Membantu pengguna baru memilih produk dengan cepat.

### 7.5 Promo Section

Konten:

- Banner promo aktif.
- Judul promo.
- Periode promo.
- Syarat singkat.
- CTA order/klaim.

Kriteria:

- Promo dapat diganti dengan mudah.
- Jika tidak ada promo aktif, section menampilkan pesan netral seperti "Promo terbaru akan hadir segera."

### 7.6 Outlet Section

Konten:

- Search input kota/area.
- Daftar outlet.
- Informasi outlet: nama, alamat, jam buka, link map.
- CTA "Buka Maps".

Kriteria:

- Untuk MVP, data outlet dapat menggunakan data statis.
- Untuk versi lanjutan, data outlet dapat dihubungkan ke CMS atau API.

### 7.7 About Section

Konten:

- Ringkasan brand.
- Nilai utama: kualitas teh, variasi rasa, custom topping, dan pengalaman yang fun.
- Statistik opsional: jumlah outlet, jumlah varian menu, kota jangkauan.

Catatan:

- Angka statistik harus memakai data valid dari brand. Jika belum tersedia, jangan tampilkan angka palsu.

### 7.8 Contact / Partnership Section

Konten:

- Form nama, email, nomor telepon, kota, kebutuhan/pesan.
- Dropdown tipe inquiry: General, Partnership, Event, Customer Feedback.
- CTA "Kirim Pesan".

Kriteria:

- Validasi input wajib.
- Tampilkan state sukses dan gagal.
- Untuk MVP, form dapat menggunakan mailto/form service. Untuk production, gunakan backend/API.

### 7.9 Footer

Konten:

- Logo.
- Quick links.
- Social media.
- Kontak resmi.
- Copyright.

## 9. Non-Functional Requirements

### 9.1 Performance

- FCP < 1.5 detik pada koneksi cepat.
- LCP < 2.5 detik.
- Total halaman awal < 1.5 MB setelah optimasi.
- Lazy load untuk gambar di bawah fold.
- Gunakan format gambar modern seperti WebP/AVIF bila memungkinkan.

### 9.2 Accessibility

- Kontras teks minimal WCAG AA.
- Semua gambar informatif memiliki alt text.
- Semua tombol dapat diakses keyboard.
- Focus state terlihat.
- Navbar mobile dapat ditutup dengan tombol dan tombol Escape bila memungkinkan.

### 9.3 SEO

- Title dan meta description unik.
- Open Graph image.
- Semantic HTML.
- Structured data Organization dan LocalBusiness bila data valid tersedia.
- Sitemap dan robots untuk production.

### 9.4 Responsiveness

Breakpoints:

- Mobile: 360px-767px.
- Tablet: 768px-1023px.
- Desktop: 1024px ke atas.

Kriteria:

- Tidak ada teks yang terpotong.
- CTA tetap mudah ditekan di mobile.
- Layout produk tetap stabil saat filter digunakan.

## 11. Design Direction

### 11.1 Visual Style

- Modern, playful, bersih, dan product-focused.
- Produk menjadi elemen visual utama.
- Gunakan ruang putih yang cukup agar menu mudah dipindai.
- Elemen interaksi harus terasa ringan dan cepat.

### 11.2 Warna

Gunakan warna brand secara proporsional:

- Primary Purple: `#7B2D8E`
- Deep Purple: `#4B165D`
- Accent Yellow: `#F5A623`
- Fresh Green: `#36A853`
- Rose: `#E84A7F`
- White: `#FFFFFF`
- Light Gray: `#F7F7F8`
- Text Dark: `#222222`

Catatan:

- Hindari halaman yang seluruhnya didominasi ungu. Warna aksen dan foto produk harus memberi variasi visual.

### 11.3 Tipografi

- Heading: Poppins, Montserrat, atau font sans-serif modern.
- Body: Inter, Poppins, atau system sans-serif.
- Ukuran teks harus nyaman dibaca di mobile.
- Jangan gunakan letter spacing negatif.

## 12. Tech Recommendation

### 12.1 Stack

- Framework: Next.js atau Vite + React.
- Language: TypeScript.
- Styling: Tailwind CSS atau CSS Modules.
- Icons: Lucide React.
- Animation: CSS transition atau Framer Motion untuk interaksi ringan.
- Deployment: Vercel, Netlify, atau hosting static.

### 12.2 Data

Untuk MVP:

- Produk, promo, dan outlet menggunakan file data lokal.

Untuk production:

- Data produk, promo, dan outlet menggunakan CMS/API.

Tools:

- Google Analytics 4.

- Vercel Analytics atau Plausible sebagai alternatif ringan.

## 14. Acceptance Criteria

- Landing page dapat dibuka dengan baik di mobile, tablet, dan desktop.
- Navbar dapat digunakan untuk navigasi antar section.
- Hero menampilkan visual produk dan dua CTA utama.
- Menu dapat difilter berdasarkan kategori.
- Promo aktif tampil dengan informasi periode.
- Outlet dapat dilihat dan memiliki link maps.
- Form contact memiliki validasi.
- Tidak ada layout shift besar saat halaman dimuat.
- Lighthouse Performance dan Accessibility minimal 90.
- Semua link CTA mengarah ke URL yang valid atau placeholder yang jelas untuk environment staging.

## 15. Risiko dan Mitigasi

| Risiko | Dampak | Mitigasi |
| --- | --- | --- |
| Aset brand belum tersedia | Visual kurang akurat | Gunakan placeholder sementara dan tandai sebagai aset yang harus diganti |
| Data promo cepat berubah | Informasi kadaluarsa | Simpan promo sebagai data terpisah agar mudah diganti |
| Data outlet banyak | Halaman berat | Tampilkan subset awal dan gunakan search/filter |
| Link order berbeda per kota | CTA membingungkan | Gunakan landing order aggregator atau pilih partner utama |
| Penggunaan brand tanpa izin | Risiko legal | Pastikan semua aset dan klaim mendapat persetujuan resmi |
